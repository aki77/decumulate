# 積立・取り崩しシミュレーター

積立 → 取り崩しの資産推移をモンテカルロ法（5,000パス）で評価する、Vue3 + TypeScript のシミュレータです。すべての計算はブラウザ内で完結し、データは外部送信されません。

参考: [asset-melt.party](https://asset-melt.party/) / [hiroppy/mf-dashboard](https://github.com/hiroppy/mf-dashboard/tree/main/apps/simulator)

## ローカルで開く

```sh
pnpm install
pnpm dev
```

ブラウザで `http://localhost:5173/` を開きます。

## GitHub Pages へのデプロイ

リポジトリ Settings → Pages → Build and deployment の Source を **GitHub Actions** に設定してください。`main` ブランチに push すると、`.github/workflows/deploy.yml` が自動でデプロイします。

## 機能

### 取り崩し戦略（5 モード）

- **月額固定**: 毎月一定額を引き出す
- **年率指定（Trinity Study）**: 初期資産の年率 X% を固定月額に換算
- **年率×リスク資産**: 毎年リスク資産残高に年率をかけて月額を再評価
- **Guyton-Klinger ガードレール**: 取り崩し年率が初期値の ±20% を逸脱したとき月額を ±10% 自動調整。下落相場での支出抑制と好況時の過剰引出を防ぐ動的戦略
- **DIE WITH ZERO（ゼロ着地）**: No-Go 期月額（床）と Slow-Go 係数・境界年齢を入力し、想定寿命時に残高が目標額へ着地する Go-Go 期月額を自動逆算。MC p50（実質値）ベースで逆算するため過大推奨を防ぐ

年率モードでは月額下限／上限（インフレ連動）を任意設定可能。

### 口座別・税制対応

- **NISA / 特定口座 / 防衛資産 / iDeCo** の 4 口座を個別管理し、取り崩し・リバランス時に口座ごとの課税ルールを適用
- NISA 年間・生涯枠の自動追跡、積立時は NISA 枠優先。特定口座から NISA への年初一括振替にも対応。夫婦モードで枠を 2 倍に拡張
- iDeCo: 初期残高・月次拠出・拠出終了年・一時金＆年金受取比率を設定可能。受取時は退職所得控除・公的年金等控除を加味した簡易税計算
- 公的年金（繰上げ -0.4%/月・繰下げ +0.7%/月）と他収入の控除

### リスク管理

- **防衛資産リバランス**: 乖離閾値を超えたとき自動リバランス。下落時スキップオプションあり
- **防衛比率グライドパス（Bond Tent）**: 引退前後に防衛比率を高めて取り崩し開始時のバッファを確保
- **シーケンスリスク可視化**: 取り崩し開始直後 5 年間が最悪 10% に該当するシナリオを赤点線で表示

### シミュレーション・診断指標

- モンテカルロ 5,000 パス（GBM・実質値）、p10〜p90 ファンチャート
- 安心度スコア（0–100）、枯渇確率、元本割れ確率
- 最大ドローダウン（p10/p50/p90）、安全引出率（SWR）自動算出
- **AI 相談用コピー**: 入力条件・主要メトリクス・MC 年次推移を Markdown 形式でクリップボードにコピー
- 商品プリセット（オルカン / S&P 500 / QQQ / 日経平均 / TOPIX）

## 結果の読み方（名目値 / 実質値）

このシミュレータは結果を 2 種類の単位で表示しています。同じ画面に両方が並ぶため、見ている数字がどちらかを意識してください。

### 名目値（インフレ調整なし）
将来そのときの円で見た金額。物価上昇は反映していません。

- 「積立・取り崩しの推移（決定論的, 名目値）」グラフ
- 「最終総資産（名目）」「総引出額（名目）」のサマリーメトリクス
- 「決定論的（月次, 名目値）」テーブル

### 実質値（インフレ控除後の購買力）
今日の物価で換算した金額。30年後の口座残高が名目1億円でも、年2%インフレなら実質はおよそ5,500万円相当 — 後者が実質値です。

- 「モンテカルロ・シミュレーション（5,000パス, 実質値）」グラフ
- 「MC 中央値残高（実質）」「p10/p90（実質）」のサマリーメトリクス
- モンテカルロ P10〜P90 の月次テーブル（純引出・年金・他収入・運用損益すべて実質値）

### 使い分け

| 判断したいこと | 見るべき値 |
| --- | --- |
| 口座残高・相続税・取り崩し中の出金額のイメージ | 名目値（最終総資産・決定論的グラフ） |
| 老後にいまの生活水準を維持できるか | 実質値（MC p50 の月次純引出） |
| 下振れたときに生活が破綻しないか | 実質値（MC p10〜p25） |
| 上振れ時の余裕 | 実質値（MC p75〜p90） |

### 生活費との比較は「今日の感覚」のままでよい

モンテカルロの月次テーブルはすべて実質値で出力されるため、「今 月50万円で生活できている」のであれば、30年後の行に表示される純引出が **50万円を上回り続けているか** をそのまま見れば判断できます。インフレ分を頭の中で換算し直す必要はありません。

- 「引出額をインフレ調整する」**オン**: 名目では毎年引出額が増えますが、実質ベースではほぼ横ばいで表示されます（= 生活水準を維持するシナリオ）。
- 「引出額をインフレ調整する」**オフ**: 名目固定の引出は、実質ベースで年々目減りしていく形で表示されます（= 購買力が下がっていく現実が見える）。

年金開始年で純引出が下がるのは正常な挙動です。年金（実質値）が生活費の一部を埋めるため、資産から引き出す分が減ります。

### 年率モード時の月額下限／上限

「年率（%）」「年率×リスク資産」モードでは、計算上の月額がどれだけ小さく・大きくなっても受け入れます。暴落時に生活費が突然半減したり、好調時に取り崩しすぎたりするのを防ぐため、

- **月額下限（万円, 任意）**: 計算月額がこの値を下回ったら、この値まで引き上げる
- **月額上限（万円, 任意）**: 計算月額がこの値を上回ったら、この値で抑える

を任意で指定できます。両方とも「今日の購買力」基準で入力し、シミュレーション内では毎年インフレ率分だけ自動で増えるため、実質価値が一定に保たれます。

- 下限を指定すると暴落時にも生活費を確保できる一方で資産枯渇リスクが上がります。資産がほぼ枯渇したときは下限を維持できず資産残全額を引き出します。
- 上限を指定すると好調時の取り崩しを抑えて将来の余力を残せます。
- 両方指定して下限 > 上限になった場合は上限が優先されます（バリデーションは出しません）。

### DIE WITH ZERO（ゼロ着地）モード

「使い切って死ぬ」思想に基づき、想定寿命時の残高を目標額に着地させる Go-Go 期月額を自動逆算するモードです。

- **Go-Go 期**（取り崩し開始〜Slow-Go 境界年齢）: アクティブに使う旺盛期。最も高い月額を引き出す
- **Slow-Go 期**（境界年齢〜No-Go 境界年齢）: 外出・旅行が減り始める期。Go-Go 月額 × Slow-Go 係数で自動計算
- **No-Go 期**（No-Go 境界年齢〜想定寿命）: 床として入力した固定月額

逆算は MC の p50（実質値・中央値シナリオ）で行うため、決定論版より約 50% 枯渇を防ぎます。「想定寿命時残高」メトリクスも同じ p50 で表示されます。

### シーケンスリスク（取り崩し開始 5 年 p10）

モンテカルロ 5,000 パスのうち、取り崩し開始直後 5 年間の累積リターンが最も悪い 10% に該当するシナリオをグラフ上に**赤点線**で表示します。

- 取り崩し直後に暴落が来ると、資産が回復する前に売却が続き、同じ期待リターンでも最終残高が大きく悪化します（= シーケンスリスク）
- 赤点線がゼロに近い年齢を確認することで「最悪ケースが何歳まで持つか」を一目で判断できます
- Bond Tent やガードレール戦略の有無でこの線がどう変わるかを比較すると、各戦略の効果が可視化されます
